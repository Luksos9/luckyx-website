Correct Answer: Problem Manager clicks Re-Analyze on the Problem record Source Problem management lifecycle - ServiceNow Docs If the link fails, search Google for: ServiceNow Zurich Re-Analyze problem reopen Risk Accepted Closed state Expert Explanation In ServiceNow Problem Management Best Practice lifecycle, the Re-Analyze UI action is the only mechanism to reopen a problem.   • CCatalog Items
  • DRecord Producers
Correct Answer: Record Producers Source Record producers - ServiceNow Docs If the link fails, search Google for: ServiceNow Zurich Record Producer catalog item type task records target table Expert Explanation A Record Producer is a type of catalog item that creates a record on any specified target table when submitted. Unlike standard catalog items (which create sc_request, sc_req_item, and sc_task records), a Record Producer can create an incident, a change request, a problem, or any other table record directly.
