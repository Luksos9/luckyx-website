4.You need to send a product alert (Publication) to all customers who have purchased a specific &quot;Router Model X&quot;. You want the list of recipients to automatically update if new customers purchase this router tomorrow. Which type of Recipient List should you create?
  • AStatic Recipient List
  • BDynamic Recipient List
  • CImport Set List
  • DEntitlement List
Show full explanation
Correct Answer: Dynamic Recipient List Source Official Doc: Targeted Communications (Zurich) If the link fails, search Google for: ServiceNow Zurich Dynamic Recipient List Targeted Communications product model ownership auto-refresh Expert Explanation Dynamic Recipient Lists in CSM Targeted Communications automatically maintain their membership by evaluating a condition against customer and product records. For a product alert, the condition would be "Install Base Item Product Model = [vulnerable model name]." The "Targeted Communications Recipients Refresh" scheduled job runs daily to add new customers who acquired the product and remove customers who no longer own it. When a Publication is sent, the Dynamic Recipient List is evaluated at send time, ensuring every owner of the affected product receives the alert with no manual list maintenance.

Why the Others Are Wrong Static Recipient List: A manually curated list. Every time a new customer acquires the product or an existing owner disposes of it, someone must manually update the list. Not feasible for large, ongoing product populations. Import Set List: A data loading mechanism for importing external data into ServiceNow tables. Not related to Targeted Communications recipient management. Entitlement List: Defines the scope of support a customer has purchased (which products, which SLA tier, which support hours). Memory Tip Dynamic = the list maintains itself. You write the rule once; the system finds the people forever. New owner buys the product: added automatically. Old owner sells it: removed automatically. You never touch the list again.

The security team creates a Dynamic Recipient List with condition "Product Model = CX-7000." The list immediately populates 1,847 customers. A Publication titled "Critical Security Patch Required: CX-7000" is sent within 30 minutes. Over the next week, 23 new customers activate CX-7000 units. When a follow-up bulletin is sent, the Dynamic Recipient List automatically includes those 23 new owners without any manual update.

6.A Service Desk Manager wants to identify &quot;hot topics&quot; or emerging issues that do not yet have a specific category, such as a sudden spike in users reporting &quot;VPN slowness&quot; after a patch. Which Predictive Intelligence capability groups similar historical cases together to help visualize these trends?
  • AClassification
  • BSimilarity
  • CClustering
  • DRegression
Show full explanation
Correct Answer: Clustering Source Official Doc: Clustering (Zurich Predictive Intelligence) If the link fails, search Google for: ServiceNow Zurich Predictive Intelligence Clustering trending case topics CSM unsupervised Expert Explanation Clustering is an unsupervised Predictive Intelligence model that groups cases (or other records) based on the similarity of their text fields and shared attributes, without requiring a predefined taxonomy. The CSM "Trending Case Topics" feature uses Clustering to automatically discover emerging issue clusters: when a new firmware version ships and 50 cases mention similar error codes, Clustering surfaces this as a new cluster before any category exists for it. The manager sees the cluster name, volume, and representative cases, enabling proactive response before it becomes a major issue.

Why the Others Are Wrong Classification: A supervised model trained to predict a field value from a predefined set (category, priority, assignment group). It requires you to know the categories in advance and provide training examples for each. Similarity: A model that compares a specific record against historical records to find the most similar ones. Used for "Similar Incidents" or "Related Cases" suggestions, not for bulk topic trend discovery. Regression: Predicts a continuous numeric output (hours to resolution, predicted cost). Not applicable for grouping cases by topic.

Memory Tip Clustering = the system notices a crowd gathering before you do. You did not define what the crowd is about; the algorithm found it in the data by looking at what cases have in common.

Real-World Example A software company releases a new version of their mobile app. Within 48 hours, Clustering detects a new cluster of 73 cases, all containing language related to "login loop," "can't sign in," and "authentication error." No one defined "login loop" as a category. The Trending Topics dashboard flags the cluster. The manager immediately creates a Known Error article, shares it with all agents, and adds it to Agent Assist, deflecting the next 200 cases that would have come in about the same issue.

  • BIt creates a Parent Case and links the Request to it.
  • CIt only creates the Request; no Case is involved.
  • DIt creates a Universal Task.
Show full explanation
Correct Answer: It creates a Parent Case and links the Request to it. Source Official Doc: CSM integration with Request Management (Zurich) If the link fails, search Google for: ServiceNow Zurich CSM Request Management extension parent Case auto-created catalog request Expert Explanation The CSM Extension for Request Management (plugin: com.sn_cs_sm_request) installs a business rule called "Create Case For Request." When a customer submits a Service Catalog Request (REQ) directly through the portal without already being in the context of an open case, this rule fires and automatically creates a parent Case linked to the Request. Memory Tip Every Request needs a CSM parent: Case > REQ > RITM > SCTASK. The Case is the customer's window into what is happening with their request, even if the work is tracked in RITM and SCTASK.

Within seconds, a Case is created titled "Support Tier Upgrade Request," linked to the REQ, which contains an RITM for "Premium Support Activation" and an SCTASK assigned to the account management team. The customer sees one Case in their portal; the fulfillment team works the SCTASK behind the scenes.

9.In a Business-to-Consumer (B2C) implementation, you need to model a family structure where a &quot;Head of Household&quot; can manage cases and view products for other family members. How is this relationship primarily defined in the data model?
  • ABy creating a parent-child relationship between two Contact records.
  • BBy populating the Household reference field on the Consumer record.
  • CBy creating a Partner account for the family.
  • DBy using the Relationship [cmn_relationship] table to link Consumers.
Show full explanation
Correct Answer: By populating the Household reference field on the Consumer record. Source Official Doc: B2C Customer Data Model (Zurich) If the link fails, search Google for: ServiceNow Zurich CSM B2C Household Consumer record family model Expert Explanation In ServiceNow's B2C data model, a Household (csm_household) is the grouping entity for a family unit. Each family member is a Consumer (csm_consumer), and their record includes a Household reference field pointing to the shared household record. This model allows one consumer to be designated the "Head of Household," granting them administrative control over shared products, service entitlements, and cases. The household structure is purpose-built for B2C and is distinct from the B2B account-contact hierarchy.

Why the Others Are Wrong Parent-child relationship between Contact records: Contact records are B2B entities (corporate contacts). The B2C model uses Consumer records and the Household table, not Contact-to-Contact parent-child links. Partner account for the family: Partner accounts represent external vendor or reseller organizations in B2B CSM. Memory Tip B2B uses Accounts with Contacts. B2C uses Households with Consumers. The Household is the B2C equivalent of the Account, and the Consumer is the equivalent of the Contact.

Real-World Example A home internet provider serves families, not businesses. When the Smith family signs up, the admin creates a Household record for "The Smith Household." Each family member (John, Mary, teenager Alex) is created as a Consumer with their Household field pointing to "The Smith Household." John is marked Head of Household. When John calls support about the shared router, the agent sees all three family members, their shared service plan, and all past cases from any member of the household.
