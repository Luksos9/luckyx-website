1.What is the primary purpose of Event Management in ServiceNow?
  • ATo manage calendar events
  • BTo process and correlate IT infrastructure events into actionable alerts
  • CTo schedule meetings
  • DTo track project milestones
Event Management processes raw events from monitoring tools, correlates them, and creates actionable alerts that can trigger incident creation or notification workflows.
2.What is the difference between an Event and an Alert?
  • AThey are the same thing
  • BEvents are raw data; Alerts are processed, correlated, and actionable
  • CAlerts come before Events
  • DEvents are for hardware; Alerts are for software
Events are raw monitoring data received from external tools. Alerts are created after events are processed, deduplicated, and correlated into meaningful, actionable items.
3.Which component receives events from external monitoring tools?
  • AThe CMDB
  • BThe MID Server
  • CThe Service Catalog
  • DThe Knowledge Base
The MID Server receives events from external monitoring tools via connectors and forwards them to the ServiceNow instance for processing.
4.What is Event Correlation in ServiceNow?
  • ALinking events to user profiles
  • BGrouping related events together to reduce noise and identify root causes
  • CSorting events by date
  • DAssigning events to support groups
Event Correlation groups related events together using correlation rules, reducing alert noise and helping identify the root cause of infrastructure issues.
5.What is the function of an Alert Management Rule?
  • ATo delete old events
  • BTo define automated actions when alerts meet certain conditions
  • CTo create new monitoring tools
  • DTo manage user passwords
Alert Management Rules define automated actions (like creating incidents, sending notifications, or running workflows) that execute when alerts match specified conditions.
