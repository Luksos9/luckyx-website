11.Which of the following describes ServiceNow’s Encryption process when data is in transit? (Choose 3) (choose 3)
  • AScrambles information to prevent its decoding or use
  • BRequires an administrator to grant an encryption context to a role
  • CExcludes HR tables from system cloning
  • DPrevents access to specific tables
  • EPrevents text fields and attachments from being indexed
  • FPrevents users from entering personal information in an HR case
Show full explanation
SourceCorrect Answers:Scrambles information to prevent its decoding or use - This is the core function of encryption, which is essential for data in transit. ServiceNow uses industry-standard encryption protocols like TLS (Transport Layer Security) to protect data moving between client devices and servers.Requires an administrator to grant an encryption context to a role - ServiceNow's encryption involves encryption contexts, which are like administrative "containers" that define what data gets encrypted and who has the keys to decrypt it. Roles need to be assigned to an encryption context for its members to be able to work with encrypted data.Prevents text fields and attachments from being indexed - Encrypted fields and attachments lose their searchability once encrypted. This is a trade-off between security and convenience, as you usually can't search the content of encrypted data.Incorrect AnswersExcludes HR tables from system cloning - While you might choose to exclude sensitive HR data when cloning a ServiceNow instance, this is not directly part of the intrinsic encryption process.Prevents access to specific tables - Encryption can make data unreadable, helping to enforce access restrictions if a breach happens. However, table-level access is based on roles and permissions, not encryption itself.Prevents users from entering personal information in an HR case - This could be a policy enforced through field-level configurations or user training, but not a direct capability of encryption.Key Points about ServiceNow Encryption:Data at Rest: ServiceNow also offers strong encryption options for data at rest (stored in the database), such as Edge Encryption and database encryption.TLS: TLS is the standard protocol for protecting web communication (like when you're using the ServiceNow web interface) and is used to secure data in transit.Trade-offs: Encryption always involves trade-offs. Encrypting more data increases security but can impact performance and make it harder to search and work with that data.
