Explanation of why it is correctWhile Vulnerability Response (VR) focuses on software flaws (like missing patches or bugs), Configuration Compliance (CC) focuses on how a system is set up.It ingests data from scanners (like Qualys Policy Compliance or Tenable.io) to see if your servers follow "Hardening" standards such as CIS Benchmarks or DISA STIGs. For example, while VR checks if you have a patch for a specific exploit, CC checks if your password length is at least 12 characters or if "Guest" accounts are disabled. Because these two practices overlap in the goal of reducing risk, they share much of the same infrastructure in ServiceNow, such as Discovered Items and Assignment Rules.3)

Explanation of why others are incorrectTrusted Security Circles: This application is used for sharing Indicators of Compromise (IoC) or threat data anonymously between different organizations. It helps prioritize vulnerabilities but does not check if a server meets a benchmark.Security Incident Response (SIR): This is for reactive "firefighting." If a hacker actually breaks in, SIR is used to manage the investigation. Configuration Compliance is a proactive tool used to prevent the break-in from happening in the first place by hardening the target.4) How to remember that more easilyThink of Home Security:Vulnerability Response = Checking for Broken Windows.
