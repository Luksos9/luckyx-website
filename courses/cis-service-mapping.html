1.What should be checked in CI Identification rules to insert/update CIs with no regard to other CIs or relationships?
  • AThe Mandatory checkbox
  • BThe Independent checkbox
  • CThe Standalone checkbox
  • DThe Override checkbox
Show full explanation
Correct Answer: The Independent checkbox Source Official Doc: CI Identification and Reconciliation Engine If the link fails, search Google for: ServiceNow Zurich CI Identification rule Independent checkbox standalone insert update Expert Explanation In the CI Identification and Reconciliation Engine (IRE), each identification rule has an Independent checkbox. When checked, the IRE processes that CI type without evaluating relationships to other CIs first. This is important for CI classes that do not depend on a host or container CI for their identity, such as top-level infrastructure devices. Without this flag, IRE may delay or fail CI creation while waiting for a dependent parent CI to be identified first.

Why the Others Are Wrong The Mandatory checkbox: Mandatory applies to individual identifier entries within a rule, indicating that identifier must be present for a match. It does not govern whether the CI is processed independently of relationships. The Standalone checkbox: Not the correct term used in ServiceNow CI Identification rule forms. The Override checkbox: Not a field on CI Identification rule records for this purpose.

Memory Tip "Independent" means the CI stands alone. It does not need a parent CI identified before it can be inserted or updated. Think of it as a self-employed CI: it files its own paperwork, no employer (parent) required. UI Navigation Path: All > Configuration > CI Class Manager > [Select Class] > Identification What to look for: The "Independent" checkbox on the identification rule form for the selected CI class.

Real-World Example A firewall CI type has the Independent checkbox enabled. When Discovery finds a Palo Alto firewall, IRE inserts it directly into cmdb_ci_firewall without waiting for a related network device parent. Without this flag, IRE might stall looking for a dependent container that does not exist.

5.What is true about the execution of connection sections?
  • AOnly the first matching connection section runs
  • BConnection sections run before identification sections
  • CAll connection sections execute after a successful identification section
  • DConnection sections only run if the CI is new
Show full explanation
Correct Answer: All connection sections execute after a successful identification section Source Official Doc: Service Mapping pattern sections If the link fails, search Google for: ServiceNow Zurich connection sections all execute after identification succeeds Expert Explanation Connection sections map the dependencies of an identified CI: the load balancers in front of it, the databases behind it, the message queues it communicates with. Because a complex application can have many types of dependencies, multiple connection sections co-exist in a single pattern. When identification succeeds, all connection sections execute in sequence, each discovering a different relationship type. Individual connection section failures do not stop the others; they are logged as partial map errors while the rest of the sections continue.

Why the Others Are Wrong Only the first matching connection section runs: This describes identification section behavior, not connection section behavior. For connections, all sections run after identification succeeds. Connection sections run before identification sections: This reverses the architecture. Connection sections need an identified source CI to attach relationships to; that requires identification to complete first. Connection sections only run if the CI is new: Service Mapping re-runs connection sections on every mapping execution to detect changes in dependencies, regardless of whether the CI is new or existing.

Memory Tip Identification: first one that wins, stops the rest. Connections: all of them run, every time.

Real-World Example A Tomcat pattern has four connection sections: upstream load balancer (HTTP check), downstream Oracle DB (JDBC connection string), downstream ActiveMQ (JMX check), and shared NFS mount (filesystem path). After Tomcat is identified on a server, all four connection sections execute. Three succeed and add relationships to the map. The NFS check fails because NFS is not configured on this instance, logging a partial error without blocking the other three relationships.

11.What would be the selling point for a customer to purchase Event Management? (Choose 4) (choose 4)
  • AReduce alert noise through correlation and deduplication
  • BAccelerate incident response through automated alert-to-incident creation
  • CGain visibility into service health via Service Mapping integration
  • DIntegrate with existing monitoring tools into one platform
  • EReplace all existing monitoring tools entirely
  • FEliminate the need for a CMDB
Show full explanation
Correct Answers: Reduce alert noise; Accelerate incident response; Gain visibility into service health; Integrate monitoring tools into one platform Source Official Doc: Event Management overview If the link fails, search Google for: ServiceNow Zurich Event Management business value selling points ITOM Expert Explanation ServiceNow Event Management delivers value across four proven dimensions. First, correlation and deduplication reduce alert volume by up to 90% in complex environments. Second, automated incident creation from alerts eliminates manual triage steps. Third, Service Mapping integration maps infrastructure alerts to impacted business services, enabling business-impact-aware prioritization. Fourth, multi-tool integration via out-of-box connectors (Dynatrace, Nagios, Splunk, etc.) creates a unified operational view without replacing existing tools.

Why the Others Are Wrong Replace all existing monitoring tools entirely: Event Management is an aggregation layer, not a replacement monitoring platform. Existing tools continue generating events; ServiceNow processes them. Eliminate the need for a CMDB: The opposite is true. Memory Tip Event Management is an air traffic controller, not a new airline. It does not replace the planes (monitoring tools). It organizes the airspace (events) so collisions (duplicate incidents) do not happen and every landing (alert) goes to the right runway (team).

Real-World Example A bank runs Dynatrace, Nagios, and a custom SNMP trap receiver. During a network incident, 3,000 events flood in over 10 minutes. Event Management correlates them into 4 alert groups, auto-creates one P1 incident for the network team, and shows the "Online Banking" service as degraded on the Operator Workspace. The NOC team addresses one ticket instead of hundreds.
12.Which phase best describes the flow of Service Mapping?
  • AEntry Point > Host Detection > Classification > Identification > Connection
  • BScanning > Classification > Identification > Exploration
  • CEntry Point > Scanning > Exploration > Connection
  • DClassification > Scanning > Entry Point > Identification
Show full explanation
Correct Answer: Entry Point > Host Detection > Classification > Identification > Connection Source Official Doc: Service Mapping application service discovery process If the link fails, search Google for: ServiceNow Zurich Service Mapping discovery process phases Expert Explanation Service Mapping uses a top-down approach starting from a defined Entry Point (URL, TCP host/port, or DB connection string). If the host is not already in the CMDB, Host Detection triggers first. Once the host exists, Classification identifies the device type, after which Identification matches the application via pattern identification sections. Finally, Connection sections trace dependencies to upstream and downstream CIs, building the full service map.

Why the Others Are Wrong Scanning > Classification > Identification > Exploration: "Scanning" and "Exploration" are Discovery phase names, not Service Mapping. Service Mapping does not have a standalone Exploration phase. Entry Point > Scanning > Exploration > Connection: Entry Point is correctly first, but Scanning and Exploration do not belong to the Service Mapping phase sequence. Classification > Scanning > Entry Point > Identification: The order is completely reversed. An Entry Point must exist before anything else can run.

Memory Tip Every Host Catches Its Connection. Entry point, Host detection, Classification, Identification, Connection.

Real-World Example A team maps their e-commerce platform. The entry point is https://shop.example.com:443. Service Mapping detects the Apache web server host, classifies it as a Linux server, identifies Apache Tomcat on port 8080 via pattern matching, then follows connections to discover a backend Oracle DB on port 1521, completing the service map.

The Service Mapping plugin (com.snc.service-mapping) must be active.
